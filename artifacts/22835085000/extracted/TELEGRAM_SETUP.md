# Telegram Notification Setup Guide

## Overview

Dellmology Pro now includes real-time Telegram notifications for trading alerts, market analysis, and broker activity. Receive alerts on your phone whenever trading opportunities arise.

## Features

### Alert Types

1. **Trading Signals** 🟢
   - BUY/SELL signals with confidence levels
   - Triggered when UPS score crosses thresholds
   - Cooldown: 5 minutes per symbol

2. **Market Analysis** 📈
   - Regime changes (Uptrend → Downtrend, etc)
   - Volatility spike alerts
   - Technical breakdown notifications

3. **Broker Activity** 🐋
   - Whale accumulation detected (Z-Score > 2.5)
   - Large broker position changes
   - Abnormal volume patterns

4. **Wash Sale Detection** ⚠️
   - Suspicious volume patterns identified
   - Market manipulation warnings
   - Integrity alerts

5. **AI Screener Results** 🤖
   - Daytrade candidates (high volatility)
   - Swing trade opportunities (whale accumulation)
   - Daily AI recommendations

6. **Backtest Reports** 📊
   - Strategy performance summaries
   - Win rate and profitability metrics
   - Sharpe ratio calculations

## Setup Instructions

### Step 1: Create Telegram Bot

1. Open Telegram and search for `@BotFather`
2. Send `/start`
3. Send `/newbot`
4. Enter bot name (e.g., "Dellmology Pro")
5. Enter bot username (e.g., "dellmology_pro_bot")
6. Copy the **Bot Token** (looks like: `123456787:ABCDefGHIjklMNOpqrSTUvwxyz`)

### Step 2: Get Your Chat ID

1. Create a new group or use existing chat
2. Add your bot to the chat
3. Send a message in the chat
4. Visit: `https://api.telegram.org/bot<YOUR_BOT_TOKEN>/getUpdates`
5. Find your chat ID in the response (looks like: `-987654321` for groups)
6. Or get personal chat ID by sending `/start` to the bot directly

### Step 3: Configure Environment

Create `.env` file in project root:

```bash
# Telegram Configuration
TELEGRAM_BOT_TOKEN=123456787:ABCDefGHIjklMNOpqrSTUvwxyz
TELEGRAM_CHAT_ID=-987654321

# ML Engine Configuration
ML_ENGINE_URL=http://localhost:8001
ML_ENGINE_KEY=your-api-key-for-telegram-service
```

### Step 4: Start Telegram Service

```bash
# With Docker Compose (includes Python service)
docker-compose up -d

# Or manually (requires Python 3.9+):
cd apps/ml-engine
pip install -r requirements.txt
python telegram_service.py
```

### Step 5: Configure Alerts in UI

1. Go to Settings → Telegram Alerts
2. Toggle alert types you want to receive
3. Click "Save Settings"

## Alert Cooldown

To avoid spam, alerts have a **5-minute cooldown** per symbol per alert type. This means:
- BBCA BUY signal sends, then next BBCA BUY won't send for 5 min
- But BBCA SELL signal can send immediately
- Another stock's signals aren't affected

## Troubleshooting

### "Not Connected" in Settings
- Check `.env` file has `TELEGRAM_BOT_TOKEN` and `TELEGRAM_CHAT_ID`
- Verify bot token is correct
- Ensure bot is added to the chat
- Restart telegram_service.py

### Alerts Not Being Sent
1. Check bot is still in the chat (if group)
2. Verify chat ID in `.env` (should start with `-` for groups)
3. Check logs: `docker logs dellmology-ml-engine`
4. Confirm alert type is enabled in Settings

### Wrong Chat Receiving Messages
- Double-check chat ID in `.env`
- Get fresh chat ID using @BotFather method above
- Restart docker or service

### Rate Limited
- Telegram allows ~30 messages per second
- Our 5-minute cooldown prevents most issues
- If still limited, increase cooldown value in `telegram_notifier.py`

## Advanced Configuration

### Custom Cooldown Period

Edit `telegram_notifier.py`:
```python
self.alert_cooldown = 600  # 10 minutes instead of 5
```

### Alert Chain

Combine condition triggers in `alert_trigger.py`:
- UPS > 80 AND Whale detected → STRONG_BUY
- UPS < 30 AND Downtrend → STRONG_SELL
- Regime change UP + Volume spike → BUY

### Batch Sending

For multiple stocks, alerts aggregate and send batch report:
```python
# In alert_aggregator.py
batch_window = 300  # Collect for 5 minutes
```

## Production Deployment

For production use:

1. **Use managed Telegram API endpoint** (not `api.telegram.org`)
2. **Add request signing** for security
3. **Implement database logging** of all alerts sent
4. **Setup alerting** if Telegram service goes down
5. **Rate limit backend** to prevent abuse
6. **Use webhook** instead of polling for updates
7. **Encrypt credentials** in environment variables
